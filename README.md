# egress-rules

Public host for **ANAI Little Snitch egress rule subscriptions**, generated by the
`mac-sec` security stack (`egress/feeds/`). Little Snitch subscribes to these over
`raw.githubusercontent.com` (trusted-cert HTTPS).

| File | What it is |
|------|-----------|
| `anai-malicious.lsrules` | Malicious-only **DENY** blocklist from abuse.ch Tier-1 feeds (Feodo C2 IPs, ThreatFox `botnet_cc`, URLhaus online malware hosts) with FireHOL Tier-3 corroboration. Self-egress + multi-tenant/compromised-legit excluded. |
| `anai-self-egress.lsrules` | High-priority **ALLOW** seed for the agent's own egress, so a feed false-positive never blocks legitimate work. |

**Posture:** observe-only — the deny set is malicious-only; no default-deny. These
lists contain only public threat intelligence plus common developer-tool
destinations. No secrets. Regenerated and republished on a schedule.

Subscribe in Little Snitch via `x-littlesnitch:subscribe-rules?url=<encoded raw url>`.
